後端能力培養(一) - 實作簡單登入系統

Sep 22, 2017

  其實自己在開始寫前端網頁之前，我是先加入一個後端能力培養的讀書會(一樣是胡立老師開的)，他不像前面的課程，有一堂一堂作業給大家，接著有YOUTUBE講解。讀書會顧名思義就是放給大家去討論，胡立有列了方向給大家一步一步去學，剩下就是自己去上網找資源。
  前陣子把前端網頁學了一個進度後，決定在這個小空檔，寫一個簡單登入系統出來，之後在慢慢繼續加功能上去，最後可以配合上我前端寫出來的網頁。

在開始寫這個登入系統之前，我給自己5個大步驟

  1. 找免費工具把伺服器架好。
  2. 把前端網頁刻出來。
  3. 利用Javascript把錯誤訊息顯示出來，以及切換Sign up/Log in。
  4. 在mySQL建立資料庫，前端用ajax跟後端做連接，並且後端連接資料庫。
  5. Sign up -> 存入資料庫，Log in -> 資料驗證。

  這也是為甚麼我決定在寫登入系統之前，決定先去學前端網頁和javascript，因為畢竟還是需要這些工具我才能把網頁架構出來以及資料處理，像是處理錯誤訊息上我需要用到javascript，前端傳訊息給後端我要利用jquery的ajax。

1. 找免費工具把伺服器架好

   大家最推的就是XAMPP，XAMPP是一套整合Apache + MariaDB + PHP + Perl，操作簡單，跨平台與開放原始碼的免費軟體，也沒什麼懸念我就直接選這個， 相關的設定和怎麼用網路上非常多資源，我就不細談。

2. 把前端網頁刻出來

   有了之前的基礎其實要寫出來就沒有太困難了，我一樣利用flex去做排版，比較麻煩的是希望把它弄得美觀一點，雖然我可能沒有太多藝術天分
還是希望它好看一點。所以在選色彩和比例上花了不少時間。

3. 利用Javascript把錯誤訊息顯示出來，以及切換Sign up/Log in

   這部分我就花了點時間，主要分成做兩件事
(1) 我需要可以利用左下角的文字做Sign up/Log in的toggle
方法就是判斷現在顯示的是甚麼，就換成另一個。 那因為Sign up多了email需要輸入，所以需要來回調整高度。

(2) 把錯誤訊息顯示在Log in和username的中間
其實跟上面方法很像，我用js去判斷如果使用者沒有輸入username我應該顯示甚麼訊息? 如果輸入的長度太短，我又應該顯示? 去做這些處理之後，當有錯誤訊息來的時候，就多一個<div> show在中間，之後一樣調整高度。因為需要避免每次錯誤訊息來都一直增加高度，所以需要用變數hasLengthen去做控制。

最後顯示錯誤的效果就會是這樣

4. 在mySQL建立資料庫，前端用ajax跟後端做連接，並且後端連接資料庫

   之前其實沒有使用過DATABASE的經驗，只懂一點SELECT, WHERE這些簡單指令，所以就算是自我學習吧! 我建立一個table叫做userdata，並有四個值id,username,password,email ， id的部分設成primary key以及AUTO_INCREMENT，primary key作用就是讓每一筆都是唯一的。

  再來就是用ajax連接後端，有了之前串接twitchAPI的經驗，這次實作就不難了，只是這次改用POST。

POST 跟 GET 的差別? HTTP Methods: GET vs. POST，這裡有蠻詳細的比較
我這裡做簡單比較，大家可以想像GET就是明信片，所以東西都寫在上面，內容會被看光，而且有內容版面限制。 POST就像是裝在信封裡的資料，信封裡的東西你看不到，你也可以裝很多資料在裡面
GET

  1. Data會顯示在URL上
  2. 因為顯示在URL上所以有長度限制
  3. 比較不安全，因為資料直接暴露在URL上，所以password或是機密的資料絕對不能用GET
  4. Get requests會留在歷史紀錄中
  5. can be cached，can be bookmarked

POST

  1. Data並不會顯示在URL上
  2. 沒有長度限制
  3. 在傳輸機密資料的時候比較安全
  4. Post requests不會留在歷史紀錄中
  5. 不會被cached，不能被bookmarked

最後就是去連接Database，因為我在自己本機測試，所以servername就是localhost:3306，3306則是XAMPP給的Default Port。

5. Sign up -> 存入資料庫，Log in -> 資料驗證

  之前雖然學過簡單的php，不過也都忘光光。 所以這次也可以說是第一次使用php寫後端功能，基本上就是邊寫邊查，然後不斷的debug。 在這裡我還沒有做任何的安全性上的保護，就是完全純粹的直接存進資料庫，直接抓出來比對。
(1) sign up
概念: 當資料來了我要先問資料庫這個username有沒有已經存在database裡? 如果出現過，告訴前端這個username已經用過。 如果username沒有用過但是email使用過，一樣告訴前端這個email已經使用過了。

技術上:
$username_sql = "SELECT * FROM userdata WHERE username='$username'"; -> 代表我現在要對資料庫做的指令，我要找database中username符合我從前端拿到的username
$username_result = $conn->query($username_sql); -> 把指令丟進去得到結果
$username_result->num_rows>0 -> 如果得到的row欄位大於0，表示找的到符合條件的，username是有存在在資料庫中
$return['error_msg'] = "Username has been used !"; -> 存取錯誤訊息
echo json_encode($return,JSON_PRETTY_PRINT); -> 用json格式回傳給前端

概念: 如果都沒用過就存進去資料庫。

技術上:
$sql = "INSERT INTO userdata (username, password, email) VALUES ('$username', '$password', '$email')"; -> 新增資料的指令
後面就是一些我自己想達到效果的回傳資料

(2) Log in
概念: 當資料來了我要先問資料庫這個username有沒有已經存在database裡? 如果沒有出現過，告訴前端這個username不存在。如果有存在，我就去比對password一不一樣，如果一樣就登入成功，不一樣就失敗。

技術上:
$username_row=mysqli_fetch_assoc($username_result); -> 利用這個function我可以得到從databse抓到資料的array，這樣我就可以利用$username_row[‘password’]，得到password的值

這邊實作暫時非常簡單，一些密碼上的儲存應該做加工，我都還沒有實作。 還有資安上的一些問題，我也還未處理，不過這也是我之後的功課就是了! 跌跌撞撞也終於是完成簡單的登入系統，有小小的成就感，期待之後為它加上更多功能!
